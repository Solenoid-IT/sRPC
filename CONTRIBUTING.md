# Contributing to sRPC™

First of all, thank you for showing interest in improving the **Solenoid Remote Procedure Call (sRPC)** protocol. Our goal is to create a fast, action-oriented, and developer-friendly standard for the modern web.

By contributing to this project, you help define the future of sRPC.

## 🚀 How to Contribute

### 1. Reporting Bugs
- Use the **GitHub Issues** tab.
- Provide a clear description of the bug and steps to reproduce it.
- Specify if the bug is in the protocol specification or in the SDK implementation.

### 2. Suggesting Protocol Changes
Since sRPC is moving towards formal standardization:
- Any change to the core protocol (headers, error codes, FLUID methods) must be discussed in a **GitHub Discussion** or Issue first.
- Significant changes should be reflected in the `draft-solenoid-srpc-xx.mkd` file (the RFC draft).

### 3. Submitting Pull Requests
- Create a feature branch from `main`.
- Ensure your code follows the existing style (TypeScript for the SDK).
- Add the standard Apache 2.0 copyright header to any new files:
  `Copyright © 2026 Solenoid-IT. Licensed under Apache 2.0.`

We are committed to providing a welcoming and inspiring community. Please be respectful and constructive in all interactions.
